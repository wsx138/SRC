# 文件上传漏洞修复报告

## 基本信息

| 项目 | 详情 |
|------|------|
| **项目名称** | 用户管理系统 (Flask Web Application) |
| **漏洞类型** | 文件上传漏洞 (Unrestricted File Upload) |
| **CWE 编号** | CWE-434: Unrestricted Upload of File with Dangerous Type |
| **OWASP 映射** | A04:2021 – Insecure Design |
| **代码仓库** | https://github.com/wsx138/SRC |
| **修复日期** | 2026-07-21 |

---

## 一、漏洞概述

### 影响功能

| 路由 | 漏洞位置 |
|------|---------|
| `/upload` | 头像上传功能 |

### 漏洞详情

修复前的上传功能存在以下 4 个安全问题：

| # | 漏洞 | 说明 |
|---|------|------|
| 1 | 无文件类型校验 | 攻击者可上传 `.py`、`.php`、`.exe` 等任意类型文件 |
| 2 | 使用原始文件名 | 攻击者可通过 `../../../etc/passwd` 进行路径穿越 |
| 3 | 文件存储在 `static/` 下 | 上传的文件可直接通过 URL 访问/执行 |
| 4 | 无文件名冲突处理 | 同名文件会上传会直接覆盖已有文件 |

---

## 二、修复前代码

```python
@app.route("/upload", methods=["GET", "POST"])
def upload():
    if request.method == "POST":
        file = request.files.get("avatar")
        if file and file.filename:
            # 直接使用用户提供的原始文件名
            upload_dir = "static/uploads"
            os.makedirs(upload_dir, exist_ok=True)
            filepath = os.path.join(upload_dir, file.filename)  # 路径穿越风险
            file.save(filepath)
            file_url = f"/{upload_dir}/{file.filename}"
            return render_template("upload.html", success=True, ...)
```

### 攻击场景

**场景 1：上传 WebShell**
```
上传文件: shell.php（内容 <?php system($_GET['cmd']); ?>）
访问 URL: http://127.0.0.1:5000/static/uploads/shell.php?cmd=whoami
结果: 服务器上执行任意命令
```

**场景 2：路径穿越覆盖系统文件**
```
上传文件名: ../../app.py
结果: 覆盖应用源码，导致服务崩溃或被篡改
```

**场景 3：上传恶意脚本**
```
上传文件: evil.py
访问 URL: http://127.0.0.1:5000/static/uploads/evil.py
结果: Flask 服务器执行恶意 Python 代码
```

**场景 4：文件名冲突覆盖**
```
用户 A 上传: avatar.png
用户 B 上传: avatar.png
结果: B 的文件覆盖 A 的文件
```

## 三、修复方案

### 3.1 扩展名白名单校验

```python
ALLOWED_EXTENSIONS = {"png", "jpg", "jpeg", "gif", "webp", "bmp"}

def _allowed_file(filename):
    if "." not in filename:
        return False
    ext = filename.rsplit(".", 1)[1].lower()
    return ext in ALLOWED_EXTENSIONS
```

只允许 6 种常见图片格式，其他类型（`.py`、`.php`、`.exe`、`.html` 等）一律拒绝。

### 3.2 文件名安全处理

```python
from werkzeug.utils import secure_filename

safe_name = secure_filename(file.filename)
```

`secure_filename()` 会：
- 移除路径穿越字符（`../`、`..\\`）
- 移除斜杠和反斜杠
- 只保留字母、数字、下划线、点、横线
- 返回 `""` 表示文件名完全无效

**示例**：
| 输入 | 输出 |
|------|------|
| `../../etc/passwd` | `etc_passwd`（路径穿越被移除） |
| `shell.php` | `shell.php` |
| `avatar.png` | `avatar.png` |

### 3.3 UUID 重命名存储

```python
unique_name = f"{uuid.uuid4().hex}.{ext}"
```

使用 `uuid.uuid4().hex` 生成 32 位随机十六进制字符串作为存储文件名，彻底避免：
- 文件名冲突覆盖
- 原始文件名中的潜在风险字符

### 3.4 存储目录隔离

```python
UPLOAD_DIR = "data/uploads"  # 不在 static/ 目录下

@app.route("/uploads/<path:filename>")
def uploaded_file(filename):
    return send_from_directory(UPLOAD_DIR, filename)
```

- 文件存储在 `data/uploads/`（非 `static/` 目录），无法直接通过 Web 服务器执行
- 使用 Flask 的 `send_from_directory()` 安全提供文件访问，内置路径穿越防护

## 四、修复后代码

```python
import uuid
from werkzeug.utils import secure_filename

ALLOWED_EXTENSIONS = {"png", "jpg", "jpeg", "gif", "webp", "bmp"}
UPLOAD_DIR = "data/uploads"

def _allowed_file(filename: str) -> bool:
    if "." not in filename:
        return False
    ext = filename.rsplit(".", 1)[1].lower()
    return ext in ALLOWED_EXTENSIONS

@app.route("/uploads/<path:filename>")
def uploaded_file(filename):
    return send_from_directory(UPLOAD_DIR, filename)

@app.route("/upload", methods=["GET", "POST"])
def upload():
    if not session.get("username"):
        return redirect("/login")

    if request.method == "POST":
        file = request.files.get("avatar")
        if not file or not file.filename:
            return render_template("upload.html", error="请选择一个文件")

        # 1. 扩展名白名单校验
        if not _allowed_file(file.filename):
            return render_template("upload.html", error="不支持的文件类型，仅允许上传图片文件")

        # 2. 安全处理文件名
        safe_name = secure_filename(file.filename)
        if not safe_name:
            return render_template("upload.html", error="文件名无效")

        # 3. UUID 重命名
        ext = safe_name.rsplit(".", 1)[1].lower() if "." in safe_name else ""
        unique_name = f"{uuid.uuid4().hex}.{ext}"

        # 4. 保存到 non-static 目录
        os.makedirs(UPLOAD_DIR, exist_ok=True)
        filepath = os.path.join(UPLOAD_DIR, unique_name)
        file.save(filepath)

        file_url = f"/uploads/{unique_name}"
        return render_template("upload.html", success=True,
                               file_url=file_url, filename=safe_name)

    return render_template("upload.html")
```

## 五、修复前后对比

| 安全维度 | 修复前 | 修复后 |
|----------|--------|--------|
| **文件类型** | 无限制，可上传任意类型 | 白名单：仅 6 种图片格式 |
| **文件名处理** | 使用原始文件名 | `secure_filename()` + UUID 重命名 |
| **存储位置** | `static/uploads/`（可直接执行） | `data/uploads/`（非 Web 可达） |
| **路径穿越防护** | 无 | `secure_filename()` + `send_from_directory()` |
| **文件名冲突** | 同名覆盖 | UUID 保证唯一 |
| **文件访问方式** | 直接 URL 访问 | 通过 `/uploads/<filename>` 路由安全提供 |

## 六、修复验证

### 测试 1：上传非图片文件

```
上传: shell.php → 预期: "不支持的文件类型，仅允许上传图片文件"
上传: evil.py   → 预期: "不支持的文件类型，仅允许上传图片文件"
上传: test.exe  → 预期: "不支持的文件类型，仅允许上传图片文件"
上传: avatar.png → 预期: "上传成功！"
```

### 测试 2：路径穿越攻击

```
上传文件名: ../../etc/passwd
secure_filename 处理后: etc_passwd
结果: 文件保存在 data/uploads/etc_passwd，不会覆盖系统文件
```

### 测试 3：文件名冲突

```
用户 A 上传 avatar.png → 存储为: a1b2c3d4...png（32 位 UUID）
用户 B 上传 avatar.png → 存储为: e5f6g7h8...png（不同的 UUID）
结果: 两个文件不会互相覆盖
```

## 七、总结

通过 4 项措施全面加固了文件上传功能：
1. **扩展名白名单** — 拒绝非图片文件
2. **`secure_filename()`** — 消除路径穿越风险
3. **UUID 重命名** — 防止覆盖攻击
4. **存储目录隔离** — 防止直接执行上传文件

修复后文件上传功能仅接受安全的图片文件，所有安全风险均已消除。
