## 一、审计摘要

对 `/upload` 头像上传功能进行代码级安全审计，共发现 **4 个安全漏洞**，全部已修复。修复手段：扩展名白名单 + `secure_filename()` + UUID 重命名 + 存储目录隔离。

---

## 二、漏洞详情

### 漏洞 #1: 无文件类型校验

| 属性 | 内容 |
|------|------|
| **严重程度** | Critical |
| **位置** | `app.py` — `/upload` 路由 |
| **影响** | 攻击者可上传 `.py`、`.php`、`.exe` 等任意类型文件 |

#### 原始代码

```python
file = request.files.get("avatar")
if file and file.filename:
    upload_dir = "static/uploads"
    os.makedirs(upload_dir, exist_ok=True)
    filepath = os.path.join(upload_dir, file.filename)
    file.save(filepath)  # 无任何类型检查
```

#### 攻击场景

1. 上传 `shell.php`（内容 `<?php system($_GET['cmd']); ?>`），访问 URL 执行任意命令
2. 上传 `evil.py`，Flask 服务器可被执行
3. 上传 `virus.exe`，诱导其他用户下载执行

#### 修复

引入扩展名白名单 `ALLOWED_EXTENSIONS = {"png", "jpg", "jpeg", "gif", "webp", "bmp"}`，非图片文件一律拒绝。

```python
def _allowed_file(filename: str) -> bool:
    if "." not in filename:
        return False
    ext = filename.rsplit(".", 1)[1].lower()
    return ext in ALLOWED_EXTENSIONS
```

### 漏洞 #2: 使用原始文件名（路径穿越）

| 属性 | 内容 |
|------|------|
| **严重程度** | Critical |
| **位置** | `app.py` — `/upload` 路由 |
| **影响** | 攻击者通过 `../../` 将文件写入任意目录 |

#### 原始代码

```python
filepath = os.path.join(upload_dir, file.filename)
# file.filename = "../../../etc/passwd"
# → os.path.join("static/uploads", "../../../etc/passwd")
# → 最终路径: "/etc/passwd"
```

#### 攻击场景

1. 文件名 `../../app.py` → 覆盖应用源码，导致服务崩溃或被篡改
2. 文件名 `../../templates/admin.html` → 替换管理页面注入恶意代码
3. 文件名 `../../data/users.db` → 覆盖数据库

#### 修复

使用 `werkzeug.utils.secure_filename()` 移除路径穿越字符。

```python
from werkzeug.utils import secure_filename

safe_name = secure_filename(file.filename)  # "../../etc/passwd" → "etc_passwd"
```

### 漏洞 #3: 文件存储在 `static/` 目录下

| 属性 | 内容 |
|------|------|
| **严重程度** | High |
| **位置** | `app.py` — `/upload` 路由 |
| **影响** | 上传文件可直接通过 URL 访问/执行 |

#### 原始代码

```python
upload_dir = "static/uploads"
file.save(os.path.join(upload_dir, file.filename))
file_url = f"/static/uploads/{file.filename}"
```

#### 问题

`static/` 是 Flask 的默认静态文件目录，所有文件可由 Web 服务器直接提供。任何上传到该目录的可执行文件（`.py`、`.php`）均可被远程执行。

#### 修复

存储目录移至 `data/uploads/`（不在 `static/` 下），通过专用路由 `send_from_directory()` 安全提供文件访问。

```python
UPLOAD_DIR = "data/uploads"

@app.route("/uploads/<path:filename>")
def uploaded_file(filename):
    return send_from_directory(UPLOAD_DIR, filename)
```

---

### 漏洞 #4: 无文件名冲突处理

| 属性 | 内容 |
|------|------|
| **严重程度** | Medium |
| **位置** | `app.py` — `/upload` 路由 |
| **影响** | 同名文件会上传会直接覆盖已有文件 |

#### 修复

使用 `uuid.uuid4().hex` 生成 32 位随机十六进制字符串作为存储文件名，同时保留原始扩展名。

```python
unique_name = f"{uuid.uuid4().hex}.{ext}"  # a1b2c3d4e5f6...png
```

## 三、完整修复后代码

```python
import uuid
from werkzeug.utils import secure_filename

ALLOWED_EXTENSIONS = {"png", "jpg", "jpeg", "gif", "webp", "bmp"}
UPLOAD_DIR = "data/uploads"

def _allowed_file(filename: str) -> bool:
    if "." not in filename:
        return False
    ext = filename.rsplit(".", 1)[1].lower()
    return ext in ALLOWED_EXTENSIONS

@app.route("/uploads/<path:filename>")
def uploaded_file(filename):
    return send_from_directory(UPLOAD_DIR, filename)

@app.route("/upload", methods=["GET", "POST"])
def upload():
    if not session.get("username"):
        return redirect("/login")

    if request.method == "POST":
        file = request.files.get("avatar")
        if not file or not file.filename:
            return render_template("upload.html", error="请选择一个文件")

        # 1. 扩展名白名单校验
        if not _allowed_file(file.filename):
            return render_template("upload.html",
                error="不支持的文件类型，仅允许上传图片文件")

        # 2. 安全处理文件名
        safe_name = secure_filename(file.filename)
        if not safe_name:
            return render_template("upload.html", error="文件名无效")

        # 3. UUID 重命名
        ext = safe_name.rsplit(".", 1)[1].lower() if "." in safe_name else ""
        unique_name = f"{uuid.uuid4().hex}.{ext}"

        # 4. 保存到 non-static 目录
        os.makedirs(UPLOAD_DIR, exist_ok=True)
        file.save(os.path.join(UPLOAD_DIR, unique_name))

        return render_template("upload.html",
            success=True, file_url=f"/uploads/{unique_name}",
            filename=safe_name)

    return render_template("upload.html")
```

## 四、修复前后对比

| 安全维度 | 修复前 | 修复后 |
|----------|--------|--------|
| **文件类型** | 无限制 | 白名单 6 种图片格式 |
| **文件名** | 原始文件名（可路径穿越） | `secure_filename()` + UUID |
| **存储位置** | `static/uploads/`（可直接执行） | `data/uploads/`（隔离） |
| **路径穿越** | 无防护 | `secure_filename()` + `send_from_directory()` |
| **文件名冲突** | 同名覆盖 | UUID 保证唯一 |
| **文件访问** | 直接 URL | 专用路由安全提供 |
