# 安全漏洞修复报告

**项目**: 用户管理系统 (Flask)  
**仓库**: https://github.com/wsx138/SRC  
**修复日期**: 2026-07-19  

---

## 漏洞清单与修复

### 1. 明文密码存储 (Critical)

| 项目 | 详情 |
|------|------|
| **文件** | `app.py` |
| **原代码** | `"password": "admin123"` / `"password": "alice2025"` |
| **风险** | 密码以明文形式存储在内存字典中，任何人获取代码即可拿到所有用户密码 |

**修复**: 引入 `werkzeug.security.generate_password_hash()` 对密码进行哈希处理，验证时使用 `check_password_hash()`，内存中不再保留原始明文密码。

```python
# 修复前
"password": "admin123"

# 修复后
"password": generate_password_hash("Admin@2025#Secure")
```

---

### 2. 弱口令 (High)

| 项目 | 详情 |
|------|------|
| **文件** | `app.py` |
| **原密码** | `admin123` / `alice2025` |
| **风险** | 极弱密码，可被暴力破解或字典攻击轻易攻破 |

**修复**: 替换为符合复杂度要求的高强度密码：
- admin: `Admin@2025#Secure`
- alice: `Alice@2025#Secure`

---

### 3. 调试注释泄露默认凭据 (High)

| 项目 | 详情 |
|------|------|
| **文件** | `templates.py/login.html` |
| **原代码** | `<!-- 调试信息 - 默认管理员账号 用户名: admin 密码: admin123 -->` |
| **风险** | 任何人查看页面 HTML 源码即可直接获得管理员账号密码 |

**修复**: 删除该调试注释，不在前端暴露任何凭据信息。

---

### 4. 密码明文展示在页面 (Medium)

| 项目 | 详情 |
|------|------|
| **文件** | `templates.py/index.html` |
| **原代码** | `<span class="info-value">{{ user.password }}</span>` |
| **风险** | 用户登录后页面直接展示密码字段，即使是哈希值也不应在前端展示 |

**修复**: 从用户信息展示列表中移除密码字段。

## 后续建议

- 将密码改用数据库存储，并使用 bcrypt / argon2 等更安全的哈希算法
- 增加登录失败次数限制，防止暴力破解
- 部署时启用 HTTPS，防止账号密码在传输过程中被窃听
